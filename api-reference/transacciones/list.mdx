## `GET /api/transacciones`

Returns all transactions that have not been soft-deleted, ordered by creation time descending (newest first). Each item includes the calculated `monto_pendiente` field.

**Authentication:** Required
**Required role:** Any authenticated user

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| Authorization | Bearer \<token\> | Yes |

## Response

### Success (200)

Returns an array of transaction objects. Each object has the following fields:

<ResponseField name="[].id" type="number">
  Transaction ID.
</ResponseField>

<ResponseField name="[].nro_reg" type="number">
  Sequential registration number.
</ResponseField>

<ResponseField name="[].concepto" type="string">
  Transaction description.
</ResponseField>

<ResponseField name="[].tipo" type="string">
  Transaction type (e.g. `venta`).
</ResponseField>

<ResponseField name="[].mesa" type="string | null">
  Service location.
</ResponseField>

<ResponseField name="[].cliente" type="string | null">
  Customer name.
</ResponseField>

<ResponseField name="[].estado" type="string">
  Status: `pendiente`, `abierto`, or `cerrado`.
</ResponseField>

<ResponseField name="[].estado_cocina" type="string">
  Kitchen status: `pendiente` or `terminado`.
</ResponseField>

<ResponseField name="[].monto_total" type="string">
  Total amount as a decimal string.
</ResponseField>

<ResponseField name="[].monto_pagado" type="string">
  Amount already paid as a decimal string.
</ResponseField>

<ResponseField name="[].monto_pendiente" type="string">
  Calculated outstanding amount (`monto_total - monto_pagado`).
</ResponseField>

<ResponseField name="[].caja_id" type="number | null">
  Associated register ID.
</ResponseField>

<ResponseField name="[].usuario_id" type="string | null">
  ID of the user who created the transaction.
</ResponseField>

<ResponseField name="[].fecha" type="string">
  Creation date (`YYYY-MM-DD`).
</ResponseField>

<ResponseField name="[].hora" type="string">
  Creation timestamp with timezone.
</ResponseField>

### Error responses

| Status | Description |
|---|---|
| 401 | Unauthorized |

## Example

```bash
curl http://localhost:3000/api/transacciones \
  -H "Authorization: Bearer $TOKEN"
```
